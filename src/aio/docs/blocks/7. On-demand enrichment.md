**7. On-demand enrichment**

Sau khi có candidate/correlation, AIOps mới enrich thêm bằng các nguồn tốn kém hoặc nhạy cảm:

```text
Jaeger
OpenSearch
Kubernetes API
```

Cụ thể:

- `Jaeger trace summary`:
  - trace ID;
  - service/operation;
  - duration;
  - error span;
  - link đến trace UI.
- `OpenSearch bounded log excerpts`:
  - log count;
  - safe excerpts;
  - link query;
  - đã redact PII/secrets/prompts.
- `Kubernetes readiness/restart/replica status`:
  - pod restart count;
  - deployment available replicas;
  - readiness;
  - rollout state.

Điểm quan trọng:

> AIOps không query Jaeger/OpenSearch liên tục. Nó chỉ query on-demand sau khi có candidate incident để kiểm soát cost và dữ liệu nhạy cảm.

Nếu Jaeger/OpenSearch fail, incident vẫn được mở. Chỉ giảm confidence hoặc ghi enrichment failure.
