**8. Incident manager**

Tất cả candidate/enrichment đi vào:

```text
Incident manager
```

Incident manager chịu trách nhiệm:

- tạo incident mới;
- deduplicate incident cũ;
- cập nhật timeline;
- tăng occurrence count;
- quản lý state;
- lưu audit;
- recovery check;
- reopen sau cooldown.

Fingerprint mặc định dựa trên:

```text
environment + detector_id + customer_flow + primary_service + likely_dependency
```

Nó không dùng timestamp, metric value, pod name, trace ID vì các giá trị đó thay đổi liên tục.
